# 演習 5: Subagentsを体験する

## 課題

### Task 1: 並列調査を依頼する
Claude Codeに以下の3つを **同時に** 調べてもらってください:

1. このリポジトリの `curriculum/` にあるファイルの一覧とそれぞれの行数
2. このリポジトリの `exercises/` にあるファイルの一覧
3. `CLAUDE.md` に書かれているフェーズの数と各フェーズの時間配分

**ポイント**: 1つずつ頼むのではなく、「以下の3つを同時に調べて」と伝えましょう。
Claude Codeが内部的にSubagentを使って並列処理するはずです。

### Task 2: 大規模検索を依頼する
「このリポジトリ全体で、Claude Codeの機能名（Read, Edit, Write, Bash, Glob, Grep, Task）が
それぞれ何回言及されているか数えて」と依頼してください。

これは広範な検索なので、Subagentが活用される場面です。

### Task 3: Subagentの動きを観察する
Task 1, 2 の実行中に、Claude Codeがどのように処理を分割しているか観察してください。
以下の点に注目しましょう:

- 何個のSubagentが起動されたか？
- 各Subagentはどんなタイプ（Explore, Bash等）か？
- 結果はどのように統合されたか？

### Task 4: 考察
Subagentを使うべき場面と、使わなくていい場面の違いを考えてみましょう。

## 確認ポイント

- [ ] 大規模検索の結果が得られた
- [ ] Subagentの動きを観察できた
- [ ] 使い分けについて考察できた

完了したら「演習5終わりました」と伝えてください。
