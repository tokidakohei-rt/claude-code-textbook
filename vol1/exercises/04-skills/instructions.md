# 演習 4: Skillを作ってみる

## 課題

### Task 1: Skillの仕組みを確認
`.claude/commands/` ディレクトリを確認し、すでに用意されているSkillがあるか調べてください。

### Task 2: レポートSkillを作る
`.claude/commands/report.md` を作成してください。

このSkillは以下の動作をするものとします:
- `$ARGUMENTS` で指定されたファイルを読む
- 以下の構造でレポートを出力する:
  1. タイトル
  2. 一言サマリー（1文）
  3. 背景と目的
  4. 要点（箇条書き3〜5個）
  5. 課題・限界
  6. 次のアクション（あれば）
- 専門用語には括弧で簡単な説明を添える
- 各セクションは3行以内に収める

### Task 3: Skillを実行する
作成した `/report` コマンドを使って、`curriculum/05-subagents.md` をレポートにしてください。

```
/report curriculum/05-subagents.md
```

「要約して」と自由に頼んだ場合と、出力の構造がどう違うか比べてみましょう。

### Task 4: オリジナルSkillを作る（チャレンジ）
自由にオリジナルのSkillを1つ作ってください。

例:
- `/translate` — ファイルを英語に翻訳する
- `/quiz` — ファイルの内容からクイズを生成する
- `/diagram` — ファイルの内容からMermaid図を生成する

## 確認ポイント

- [ ] .claude/commands/ の構造がわかった
- [ ] report Skill を作成できた
- [ ] /report を実行して構造化されたレポートが得られた
- [ ] （チャレンジ）オリジナルSkillを作れた

完了したら「演習4終わりました」と伝えてください。
