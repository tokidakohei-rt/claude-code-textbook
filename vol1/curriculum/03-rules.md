# Chapter 3: Rules - Claude Codeの振る舞いを制御する

## Rulesとは？

Rulesは、Claude Codeの動作を **プロジェクト単位・ユーザー単位で制御する設定** です。
このリポジトリの `CLAUDE.md` 自体が、まさにRulesの実例です。

## Rulesの種類と優先順位

| 種類 | ファイル | 適用範囲 | 用途 |
|-----|---------|---------|------|
| プロジェクトルール | `CLAUDE.md`（リポジトリルート） | このプロジェクト | コーディング規約、ワークフロー |
| ディレクトリルール | 各ディレクトリの `CLAUDE.md` | そのディレクトリ以下 | ディレクトリ固有のルール |
| ユーザールール | `~/.claude/CLAUDE.md` | 全プロジェクト共通 | 個人の好み、言語設定 |
| プロジェクト設定 | `.claude/settings.json` | このプロジェクト | 権限、MCP設定 |

### 読み込み順
ユーザールール → プロジェクトルール → ディレクトリルール
（後から読まれたものが優先）

## CLAUDE.md の書き方

### 効果的なCLAUDE.mdのポイント

1. **役割を明示する**: 「あなたは〇〇です」
2. **具体的なルールを箇条書きにする**: 曖昧さを排除
3. **出力形式を指定する**: フォーマット、言語、トーン
4. **禁止事項を明記する**: やってほしくないこと

### 例: このリポジトリのCLAUDE.md

```markdown
# プロジェクトルール
## あなたの役割
あなたは受講者の伴走講師です。
## ルール
- 日本語で対話する
- 演習の答えを先に出さない
- 各チャプターの流れ: 解説 → 演習 → 確認 → 次へ
```

## .claude/settings.json

プロジェクトの設定ファイルです:

```json
{
  "permissions": {
    "allow": ["Read", "Glob", "Grep"],
    "deny": []
  }
}
```

## ディレクトリごとのCLAUDE.md

大きなプロジェクトでは、ディレクトリごとに異なるルールを設定できます:

```
project/
├── CLAUDE.md              # プロジェクト全体のルール
├── frontend/
│   └── CLAUDE.md          # 「Reactコンポーネントはfunctionalで書く」
├── backend/
│   └── CLAUDE.md          # 「エラーは必ずログに出す」
└── docs/
    └── CLAUDE.md          # 「ですます調で書く」
```
