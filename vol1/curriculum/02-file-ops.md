# Chapter 2: ファイル操作を深掘りする

## Edit vs Write の使い分け

| 操作 | ツール | いつ使う？ |
|-----|--------|----------|
| 既存ファイルの一部を変更 | Edit | 大半のケース。既存コードの修正 |
| ファイルを丸ごと作成 | Write | 新規ファイル作成時のみ |
| ファイルを丸ごと上書き | Write | 構造を大幅に変えるとき |

### 重要な原則
> Claude Codeは **Edit を優先** します。既存ファイルがあるなら、Write で上書きするより Edit で部分変更する方が安全です。

## 複数ファイルの同時操作

Claude Codeは一度の指示で複数ファイルを操作できます:

```
「src/ 以下のすべての .js ファイルで、console.log を削除して」
```

内部的には:
1. Glob で対象ファイルを検索
2. Grep で console.log を含む箇所を特定
3. Edit で各ファイルを修正

## Glob パターンの基本

| パターン | 意味 |
|---------|------|
| `*.md` | カレントディレクトリの .md ファイル |
| `**/*.md` | 全階層の .md ファイル |
| `src/**/*.{ts,tsx}` | src以下の .ts と .tsx ファイル |

## Grep の使い方

| 指示例 | 内部動作 |
|--------|---------|
| 「TODO を探して」 | `pattern: "TODO"` で全ファイル検索 |
| 「.ts ファイルだけで探して」 | `type: "ts"` でフィルタ |
| 「前後3行も見せて」 | `context: 3` で周辺行を表示 |

## コンテキストウィンドウを意識する

Claude Codeには **コンテキストウィンドウ**（一度に扱える情報量）の上限があります。

- 大きなファイルを丸ごと読むと、コンテキストを圧迫する
- `/compact` で会話を圧縮できる
- 必要な部分だけ読む（`offset` と `limit` パラメータ）のがベストプラクティス
