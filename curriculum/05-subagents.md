# Chapter 5: Subagents - 並列処理とタスク分割

## Subagentsとは？

Subagent（子エージェント）は、Claude Codeが **別のエージェントを起動して並列に作業させる** 仕組みです。
内部的には `Task` ツールを使って実現されます。

## なぜSubagentsが必要か？

1つのエージェントですべてをやると:
- コンテキストウィンドウが圧迫される
- 1つずつ順番に処理するので遅い
- 複雑なタスクで混乱しやすい

Subagentsを使うと:
- **並列実行** で高速化
- 各エージェントが **独立したコンテキスト** を持つ
- メインの会話が **クリーンに保たれる**

## Subagentの種類

| タイプ | 用途 | 利用可能ツール |
|-------|------|-------------|
| `Bash` | コマンド実行 | Bash |
| `Explore` | コードベース調査 | Read, Glob, Grep等（編集不可） |
| `general-purpose` | 汎用タスク | ほぼすべて |
| `Plan` | 設計・計画 | 読み取り系のみ |

## 使い方の例

### 並列調査
```
「以下の3つを同時に調べて:
  1. src/の構造
  2. package.jsonの依存関係
  3. READMEの内容」
```
→ Claude Codeは3つのSubagentを **同時に** 起動して調査します。

### 重い処理の委譲
```
「このリポジトリ全体で、非推奨APIの使用箇所を調べて」
```
→ Exploreタイプのsubagentに委譲し、メインの会話コンテキストを汚さない。

## Subagentの動作の仕組み

```
メインエージェント（あなたとの会話）
    │
    ├── Subagent A (Explore) → 「src/の構造を調査」
    ├── Subagent B (Explore) → 「依存関係を調査」
    └── Subagent C (Bash)    → 「テストを実行」
    │
    ← 3つの結果を統合して回答
```

### ポイント
- Subagentは **メインの会話履歴を見れない**（独立したコンテキスト）
- そのため、Subagentへの指示は **自己完結** している必要がある
- 結果はメインエージェントに返され、統合される

## バックグラウンド実行

Subagentは **バックグラウンド** でも実行できます:
- 長時間かかるタスクを裏で走らせながら、別の作業を続けられる
- 結果は後から確認できる

## 実際にSubagentが使われる場面

Claude Codeは内部的に、以下のような場面で自動的にSubagentを使います:

1. **広範なコード検索**: 多数のファイルを横断する検索
2. **複数の独立した調査**: 同時に複数の質問に答えるとき
3. **プランモード**: 実装計画を立てるとき（Planタイプ）
