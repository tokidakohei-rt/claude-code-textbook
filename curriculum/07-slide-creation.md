# Chapter 7: スライド作成 - 最終アウトプット

## ゴール

`output/slides.md` に **Marp形式** のスライドを生成します。

## Marpとは？

Marp（Markdown Presentation Ecosystem）は、Markdownでスライドを作るツールです。

### 基本構文

```markdown
---
marp: true
theme: default
paginate: true
---

# スライドタイトル

---

## セクション1

- ポイント1
- ポイント2

---

## セクション2

テキストと **強調** を使えます

```

→ `---` でスライドを区切るだけ。シンプル！

## 推奨スライド構成（8-12枚）

| スライド | 内容 |
|---------|------|
| 1 | タイトル（論文名、著者、発表年） |
| 2 | 研究の背景と動機 |
| 3 | 研究の問い / 目的 |
| 4-5 | 手法の概要 |
| 6-7 | 主要な結果 |
| 8 | 図表のハイライト（あれば） |
| 9 | 考察・意義 |
| 10 | 限界と今後の展望 |
| 11 | まとめ |
| 12 | 参考文献 |

## スライドデザインのコツ

1. **1スライド1メッセージ**: 詰め込みすぎない
2. **箇条書きは5項目以内**: 多すぎると読めない
3. **キーワードを太字に**: 視線を誘導する
4. **数値は具体的に**: 「改善した」→「精度が15%向上」

## テンプレートの活用

`templates/slide-template.md` にテンプレートを用意しています。
これをベースに、論文の内容を埋めていきましょう。

## 完成したら

1. `output/slides.md` を確認
2. 内容を見直して微調整
3. Marp CLIやVS Code拡張でPDF/HTMLに変換（任意）

```bash
# Marp CLIでPDFに変換する場合
npx @marp-team/marp-cli output/slides.md --pdf
```
