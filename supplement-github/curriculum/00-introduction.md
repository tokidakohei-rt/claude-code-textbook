# Chapter 0: なぜ Git / GitHub が必要か

## Claude Code を使うなら Git は避けて通れない

Claude Code はファイルの読み書きができる強力なツールです。しかし、こんな場面で困ったことはないでしょうか？

- 「さっきの状態に戻したい」のに戻せない
- 「何をどう変更したか」がわからなくなる
- 作ったものを他の人に共有したい

**Git** と **GitHub** は、これらの問題を解決します。

## Git と GitHub の関係

よく混同されますが、別のものです。

| | Git | GitHub |
|---|---|---|
| **何か** | バージョン管理ツール | Git リポジトリのホスティングサービス |
| **どこで動く** | あなたのPC（ローカル） | インターネット上（クラウド） |
| **役割** | 変更履歴を記録する | 記録をクラウドに保存・共有する |
| **例えるなら** | セーブ機能 | クラウドのセーブデータ置き場 |

```
あなたのPC                        GitHub（クラウド）
┌──────────────┐     git push     ┌──────────────┐
│  Git で管理   │  ──────────→   │  リモート     │
│  されたファイル │  ←──────────   │  リポジトリ   │
└──────────────┘     git pull     └──────────────┘
```

## GitHub CLI（gh）とは

**GitHub CLI（`gh`）** は、GitHub の操作をターミナルから行うためのツールです。

ブラウザで GitHub を開かなくても:
- リポジトリの作成
- プルリクエストの作成・確認
- Issue の管理

がターミナルだけで完結します。

> **ポイント**: Claude Code は内部で `git` や `gh` コマンドを使います。コマンドを暗記する必要はありません。「コミットして」「プッシュして」と頼めば Claude Code がやってくれます。ただし、**何が起きているか** を理解しておくことが大切です。
