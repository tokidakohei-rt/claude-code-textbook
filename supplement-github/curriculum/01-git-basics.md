# Chapter 1: Git の基本操作

## Git の考え方 — 「セーブポイント」を作る

ゲームのセーブデータを想像してください。

- **セーブする** = `git commit`（変更を記録する）
- **セーブデータ一覧** = `git log`（履歴を見る）
- **前のセーブに戻す** = `git checkout` / `git restore`

Git は「プロジェクトのセーブポイント」を作る仕組みです。

## 最初に覚える5つのコマンド

### 1. `git status` — 今の状態を確認する

```bash
git status
```

「今どのファイルが変更されているか」を教えてくれます。**迷ったらまず `git status`** と覚えてください。

表示の読み方:

| 表示 | 意味 |
|------|------|
| `Changes not staged for commit` | 変更したけどまだセーブ対象にしていない |
| `Changes to be committed` | セーブ対象になっている（ステージング済み） |
| `Untracked files` | Git がまだ追跡していない新しいファイル |

### 2. `git add` — セーブ対象に入れる

```bash
git add ファイル名          # 特定のファイルだけ
git add curriculum/         # フォルダごと
```

「このファイルを次のセーブに含める」という操作です。Git では「**ステージング**」と呼びます。

> **なぜ2段階？**: いきなりセーブ（commit）ではなく、一度「セーブ対象の選択」（add）を挟みます。「この変更はセーブしたいけど、あの変更はまだ」という使い分けができるためです。

### 3. `git commit` — セーブする

```bash
git commit -m "ここにメッセージを書く"
```

ステージングしたファイルを **まとめて1つのセーブポイント** にします。`-m` の後のメッセージは「何をしたか」の説明です。

良いコミットメッセージの例:
```
git commit -m "Chapter 1の演習を完了"
git commit -m "レポートテンプレートの見出しを修正"
```

### 4. `git log` — 履歴を見る

```bash
git log --oneline
```

これまでのセーブポイント（コミット）の一覧が表示されます。

```
b9fd038 Vol.2追加: 業界研究レポート教材
eff7cf2 サンプル論文を同梱
610f7fa カリキュラム開始の明示的な導線を追加
94d8250 Initial commit
```

各行が1つのセーブポイントです。左の英数字は「コミットID」（セーブデータの番号）。

### 5. `git diff` — 何が変わったか見る

```bash
git diff                    # まだステージングしていない変更
git diff --staged           # ステージング済みの変更
```

ファイルのどこがどう変わったかを表示します。

## セーブの流れ（まとめ）

```
ファイルを編集
    ↓
git status          ← 何が変わったか確認
    ↓
git add ファイル名   ← セーブ対象に入れる
    ↓
git status          ← ステージングされたか確認
    ↓
git commit -m "..."  ← セーブ！
    ↓
git log --oneline    ← セーブされたか確認
```

> **ポイント**: この流れを全部自分で打つ必要はありません。Claude Code に「今の変更をコミットして」と頼めば、`git add` → `git commit` を代わりにやってくれます。ただし「何が起きているか」を知っておくと、トラブル時に自分で対処できます。

## .gitignore — 追跡しないファイルを指定する

すべてのファイルを Git で管理する必要はありません。パスワードファイルや一時ファイルは除外すべきです。

`.gitignore` ファイルに書いたパターンは Git の追跡対象から外れます:

```
# この行はコメント
.env            # 環境変数（秘密情報）
node_modules/   # 依存パッケージ（巨大）
*.pdf           # PDFファイル全般
output/         # 生成物
```

> **注意**: パスワード、APIキー、認証情報を含むファイルは **絶対に Git に入れない** でください。一度 commit すると履歴に残り、完全に消すのが非常に難しくなります。

## Claude Code に頼むとき

Git コマンドを覚えなくても、Claude Code に自然言語で頼めます:

| やりたいこと | Claude Code への頼み方 |
|------------|---------------------|
| 状態確認 | 「git の状態を見せて」 |
| セーブ | 「今の変更をコミットして」 |
| 履歴確認 | 「最近のコミット履歴を見せて」 |
| 差分確認 | 「何が変わったか見せて」 |

Claude Code は適切な git コマンドを実行してくれます。
