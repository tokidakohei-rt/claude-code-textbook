# 演習 2: GitHub とリモート操作

## 課題

### Task 1: リモートの設定を確認する

```bash
git remote -v
```

以下を答えてください:
- リモートの名前は何か？（通常は `origin`）
- 接続先の URL は何か？

### Task 2: push と pull を理解する

Claude Code に以下を頼んでください:

```
最近のコミット履歴と、リモートとの差分を見せて
```

ローカルにあってリモートにない変更、またはその逆があるか確認しましょう。

### Task 3: ブランチを体験する

1. 新しいブランチを作って切り替えます:
   ```bash
   git checkout -b practice/my-branch
   ```

2. `git branch` でブランチ一覧を確認し、現在のブランチに `*` がついていることを確認

3. 何かファイルを編集してコミットしてください（内容は自由）

4. main ブランチに戻ります:
   ```bash
   git checkout main
   ```

5. 先ほど編集したファイルが **元に戻っている** ことを確認してください

> **ここが大事**: ブランチを切り替えると、ファイルの状態もそのブランチの時点に戻ります。main の内容は安全なまま、別ブランチで自由に試行錯誤できます。

### Task 4: Claude Code でブランチ操作する（チャレンジ）

Claude Code に以下を頼んでください:

```
practice/my-branch に切り替えて、そのブランチの変更内容を見せて
```

その後:

```
main ブランチに戻って、practice/my-branch を削除して
```

Claude Code がどんなコマンドを組み合わせるか観察してください。
