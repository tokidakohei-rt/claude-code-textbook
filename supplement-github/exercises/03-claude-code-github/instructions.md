# 演習 3: Claude Code × GitHub 実践ワークフロー

## 課題

Claude Code を通じて Git / GitHub の一連のワークフローを体験します。
すべての操作を **Claude Code への自然言語の指示** で行ってください。

### Task 1: 状態確認 → 変更 → コミット → push

以下を順番に Claude Code に頼んでください:

1. **状態確認**:
   ```
   今のリポジトリの状態と、最新のコミット3件を見せて
   ```

2. **ファイル作成**:
   ```
   supplement-github/ に practice-log.md を作って。中身は「## 練習ログ」「- 演習3を実施中」と書いて
   ```

3. **コミット**:
   ```
   practice-log.md をコミットして
   ```

4. **push**:
   ```
   push して
   ```

5. GitHub のリポジトリページを開いて、ファイルが反映されたか確認してください

### Task 2: ブランチ → 作業 → PR 作成

一連の流れを Claude Code に頼みます:

1. **ブランチ作成**:
   ```
   practice/exercise-3 という新しいブランチを作って切り替えて
   ```

2. **作業**:
   ```
   practice-log.md に「- ブランチでの作業を体験」と追記して
   ```

3. **コミット & push**:
   ```
   変更をコミットして push して
   ```

4. **PR 作成**:
   ```
   main ブランチへのプルリクエストを作って。タイトルは「演習3: Git/GitHub練習」で
   ```

5. PR が作成されたら、表示された URL を開いて GitHub 上で確認してください

### Task 3: 後片付け

1. **main に戻る**:
   ```
   main ブランチに切り替えて
   ```

2. **練習ファイルの削除**:
   ```
   practice-log.md を削除してコミットして push して
   ```

3. **PR を閉じる**（マージせずに）:
   ```
   さっき作ったPRを閉じて
   ```

4. **練習ブランチの削除**:
   ```
   practice/exercise-3 ブランチをローカルとリモートの両方から削除して
   ```

### Task 4: 振り返り（チャレンジ）

Claude Code に以下を聞いてみてください:

```
今日やった Git 操作を時系列で振り返って、それぞれ何のコマンドが使われたか教えて
```

## 確認ポイント

- [ ] Claude Code を通じてコミット → push の流れを実行できた
- [ ] ブランチ作成 → 作業 → PR 作成の流れを体験した
- [ ] GitHub 上でファイルや PR を確認できた
- [ ] 後片付け（ブランチ削除、PR クローズ）ができた
- [ ] （チャレンジ）使われたコマンドを振り返って理解できた

完了したら「演習3終わりました」と伝えてください。
