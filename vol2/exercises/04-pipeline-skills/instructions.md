# 演習 4: パイプラインSkillsを体験する

## 課題

### Task 1: Skill の中身を読み比べる

以下の3つの Skill ファイルを読んで、それぞれの「入力」と「出力」を整理してください:

- `.claude/commands/research.md`
- `.claude/commands/analyze.md`
- `.claude/commands/report.md`

以下の形でまとめてみましょう:

| Skill | 入力 | 出力先 |
|-------|-----|-------|
| /research | ? | ? |
| /analyze | ? | ? |
| /report | ? | ? |

### Task 2: パイプラインを実行する

以下の順で Skill を実行してみましょう（テーマは「クラウドゲーミング」等、好きなもので OK）:

1. `/research [テーマ]`
2. 出力されたファイルを確認する
3. `/analyze output/research-[テーマ].md`
4. 出力されたファイルを確認する

各段階の出力を `/review` でチェックし、品質を確認しましょう。

### Task 3: Skill をカスタマイズする

`/research` Skill を1つだけ改良してください。

改良のアイデア:
- 調査項目に「規制・法的環境」を追加する
- 「日本語と英語の両方で検索する」ルールを追加する
- 出力フォーマットに「情報の信頼度」列を追加する

改良したら、もう一度 `/research` を実行して、出力がどう変わるか確認しましょう。

### Task 4: 自分だけのSkillを作る（チャレンジ）

パイプラインに組み込める新しい Skill を1つ作ってください。

例:
- `/competitors [テーマ]` — 競合企業を詳しく調べる Skill
- `/forecast [テーマ]` — 今後の見通しを予測する Skill
- `/summary [ファイル]` — 長いレポートをエグゼクティブサマリーにまとめる Skill

## 確認ポイント

- [ ] 3つの Skill の入出力の関係を理解できた
- [ ] パイプライン（/research → /analyze）を実行できた
- [ ] Skill を1つカスタマイズできた
- [ ] （チャレンジ）オリジナル Skill を作成できた

完了したら「演習4終わりました」と伝えてください。
