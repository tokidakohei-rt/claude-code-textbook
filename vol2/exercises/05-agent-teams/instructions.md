# 演習 5: Agent Teams を体験する

## 課題

### Task 1: 順番処理 vs 並列処理を比較する

まず、以下のタスクを **1つずつ順番に** 頼んでみましょう:

```
curriculum/02-web-research.md を要約して
```

完了を待ってから:

```
curriculum/03-hooks.md を要約して
```

次に、以下のように **まとめて** 頼んでみましょう:

```
以下の2つのファイルを並列で要約してください:
1. curriculum/04-pipeline-skills.md
2. curriculum/05-agent-teams.md

それぞれ別のファイルとして output/ に保存してください。
```

Claude Code がどのように処理を分けるか観察してください。

### Task 2: タスク分解を設計する

「日本のフィンテック業界を調査する」というタスクを、Agent Teams 向けに分解してください。

以下の形式で書き出しましょう:

```
フィンテック業界調査
├── 並列で実行できるタスク
│   ├── タスクA: [内容]
│   ├── タスクB: [内容]
│   └── タスクC: [内容]
└── 順番にやるべきタスク
    ├── 1. [内容]（上記の完了後）
    └── 2. [内容]
```

### Task 3: 並列調査を実行する

Task 2 で設計した並列タスクを、実際に Claude Code に実行させてください。

```
以下の調査を並列で実行してください:
1. [タスクA]
2. [タスクB]
3. [タスクC]

各結果は output/ に個別ファイルで保存してください。
```

### Task 4: 結果を統合する（チャレンジ）

並列調査の結果を1つのファイルに統合してください:

```
output/ にある調査結果ファイルをすべて読み込み、
内容を統合して output/integrated-fintech.md にまとめてください
```

統合結果を `/review` でチェックしましょう。

## 確認ポイント

- [ ] 順番処理と並列処理の違いを体験した
- [ ] タスク分解（並列 / 順番）を設計できた
- [ ] 並列調査を実行できた
- [ ] （チャレンジ）結果を統合してレビューを受けた

完了したら「演習5終わりました」と伝えてください。
