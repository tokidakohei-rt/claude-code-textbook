# 演習 6: 業界研究レポートを完成させる（最終課題）

## ゴール

自分が選んだ業界テーマについて、構造化された研究レポートを完成させ、レビューを通じて改善すること。

## 手順

### Step 1: テーマを決める

興味のある業界テーマを1つ選んでください（Chapter 6 で候補を参照）。

### Step 2: パイプラインを実行する

以下の順で進めてください:

```
1. /research [テーマ]
2. /review output/research-[テーマ].md
3. （必要なら追加調査）
4. /analyze output/research-[テーマ].md
5. /review output/analysis-[テーマ].md
6. （必要なら分析を修正）
7. /report output/research-[テーマ].md output/analysis-[テーマ].md
8. /review output/report-[テーマ].md
9. レビューを踏まえて改善
10. 再度 /review で最終チェック
```

### Step 3: 品質チェック

完成したレポートが以下の基準を満たしているか確認してください:

- [ ] **構成**: テンプレートの構成（7セクション）に沿っている
- [ ] **情報の質**: 数字・出典が含まれている
- [ ] **具体性**: 企業名、数値、事例が入っている
- [ ] **最新性**: 直近1-2年の情報が中心
- [ ] **読みやすさ**: 5分で全体像が掴める分量

### Step 4: レビュアーをカスタマイズする（チャレンジ）

`/review` Skill の中身を読み、自分の仕事に合わせた評価観点に書き換えてみてください。

例:
- 営業資料用: 「顧客にとってのメリットが明確か」を追加
- 社内報告用: 「上司が3分で理解できるか」を追加
- 学術用: 「引用のフォーマットが正しいか」を追加

### Step 5: ツールキットをまとめる（任意）

講座で作った Skills と設定を、他のプロジェクトでも使えるようにまとめたい場合:

```
ここで作った Skills と Hooks の設定を my-toolkit/ フォルダにまとめて、
使い方の README も作ってください
```

## 最終成果物

```
output/
├── research-[テーマ].md    ← 調査結果
├── analysis-[テーマ].md    ← 分析結果
└── report-[テーマ].md      ← 業界研究レポート（完成版）
```

## 確認ポイント

- [ ] テーマを選び、パイプラインを通して調査→分析→レポートを完成させた
- [ ] /review を使ったフィードバックサイクルを少なくとも2回実施した
- [ ] 品質チェックの基準を概ね満たしている
- [ ] （チャレンジ）/review Skill をカスタマイズした

完了したら「最終課題終わりました」と伝えてください。
