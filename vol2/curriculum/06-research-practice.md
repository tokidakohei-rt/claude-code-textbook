# Chapter 6: 業界調査の実践

## いよいよ本番

ここまでで学んだ全ての機能を使って、実際に業界調査を行います。

### 使う機能

| 機能 | 使い方 |
|------|-------|
| WebSearch / WebFetch | 最新情報の収集 |
| Skills (`/research`, `/analyze`) | 構造化された調査・分析 |
| Agent Teams | 並列調査で効率化 |
| `/review` | 各段階でのフィードバック |

## Step 1: テーマを決める

まず、調査する業界テーマを1つ選んでください。

### テーマの例

- 生成AI市場
- 電気自動車（EV）業界
- 日本のSaaS市場
- フードテック業界
- サイバーセキュリティ市場
- 再生可能エネルギー市場

もちろん、自分が興味のある業界でOKです。

### 良いテーマの条件

- Webで情報が見つかりやすい（ニッチすぎない）
- 最近動きがある（直近1-2年でニュースがある）
- 自分が少しでも興味がある（モチベーション大事）

## Step 2: `/research` で情報収集

テーマが決まったら、まず `/research` を実行します:

```
/research [あなたのテーマ]
```

例:
```
/research 日本の生成AI市場
```

調査結果が `output/research-[テーマ].md` に保存されます。

### 調査結果を確認する

保存されたファイルを開いて、以下をチェックしましょう:

- [ ] 市場規模の数字が含まれているか
- [ ] 主要プレイヤーが3社以上挙がっているか
- [ ] 出典（URL）が記載されているか
- [ ] 情報が最新か（1年以内が理想）

### レビューを受ける

```
/review output/research-[テーマ].md
```

フィードバックに基づいて、必要なら追加調査を依頼しましょう:

```
上のレビューを踏まえて、[指摘された内容] について追加で調べて、
output/research-[テーマ].md に追記してください
```

## Step 3: `/analyze` で分析

調査結果が揃ったら、分析に進みます:

```
/analyze output/research-[テーマ].md
```

分析結果が `output/analysis-[テーマ].md` に保存されます。

### 分析結果を確認する

- [ ] エグゼクティブサマリーが3行以内で書かれているか
- [ ] 成長ドライバーとリスク要因が明確か
- [ ] 「So What?」が具体的で actionable か

### レビューを受ける

```
/review output/analysis-[テーマ].md
```

## Step 4: 並列調査で深掘り（任意）

もっと深い調査をしたい場合は、Agent Teams を活用しましょう:

```
以下の3つの深掘り調査を並列で実行してください:

1. [テーマ]の主要企業トップ3の最新戦略を比較
2. [テーマ]に影響する最新の法規制や政策を調査
3. [テーマ]の海外（特に米国・中国）との比較

各結果は output/ に個別ファイルで保存してください。
```

この追加調査の結果は、次の Chapter でレポートに統合します。

## ここまでの成果物

この時点で `output/` には以下のファイルがあるはずです:

```
output/
├── research-[テーマ].md      ← 調査結果
├── analysis-[テーマ].md      ← 分析結果
└── (追加の深掘り調査ファイル)  ← 任意
```
