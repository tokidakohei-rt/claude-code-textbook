# Chapter 2: Web検索と情報収集

## なぜWebの情報が必要か

Vol.1 では手元のPDFを読むだけでした。しかし、業界研究レポートを作るには **最新の情報** が必要です。

Claude Code には Web から情報を取得する2つのツールが組み込まれています。

## WebSearch — キーワードで検索する

Claude Code に「〇〇について検索して」と頼むと、**WebSearch** ツールが使われます。

```
生成AI市場の最新動向を検索して
```

Google検索のように、キーワードに関連する最新の情報を取得してくれます。

### WebSearch のコツ

| やり方 | 良い例 | 悪い例 |
|--------|-------|-------|
| 具体的に検索する | 「日本 生成AI市場 2024年 市場規模」 | 「AI」 |
| 年を指定する | 「EV市場 2024年 トレンド」 | 「EVについて教えて」 |
| 英語も併用する | 「generative AI market size 2024」 | （日本語だけだと情報が限られる場合がある） |

> **ポイント**: Claude Code は検索結果のURLを元に情報を要約してくれます。ただし「どんなキーワードで検索するか」は、人間の指示の質に大きく左右されます。

## WebFetch — 特定のURLの内容を取得する

特定のWebページの内容を読ませたい場合は、URLを指定します。

```
https://example.com/report-2024 の内容を読んで要約して
```

**WebFetch** ツールが使われ、そのページの内容を取得して処理します。

### WebFetch の注意点

- ログインが必要なページは取得できない
- PDFや画像は直接取得できない場合がある
- ページが非常に長い場合は要約される

## MCP Server との違い

> **コラム**: 「MCP（Model Context Protocol）Server」という仕組みもあります。これは Slack、データベース、社内APIなど **Claude Code 単体ではアクセスできない外部サービス** を接続するためのものです。Web検索だけなら MCP は不要です。自分の業務で「Slackのメッセージを分析したい」「社内DBからデータを引きたい」といったニーズが出てきたら、MCP を調べてみてください。

## 信頼性の高い情報を集めるコツ

Web検索で集めた情報は玉石混交です。以下を意識しましょう:

1. **複数ソースで裏を取る**: 1つの記事だけで判断しない
2. **一次情報を優先する**: 政府統計、企業IR、学術論文 > ニュース記事 > ブログ
3. **日付を確認する**: 古い情報を最新と思い込まない
4. **数字の出典を記録する**: 後で「この数字どこから？」と聞かれたときに答えられるように

## 調査の実践フロー

業界研究では、以下の順で情報を集めるのが効率的です:

```
1. まず全体像を掴む検索（「〇〇市場 概要」）
     ↓
2. 具体的な数字を探す（「〇〇市場 市場規模 2024」）
     ↓
3. 主要プレイヤーを調べる（「〇〇業界 主要企業 ランキング」）
     ↓
4. トレンドを追う（「〇〇 最新トレンド 2024」）
     ↓
5. 課題・リスクを調べる（「〇〇業界 課題 リスク」）
```

> **ポイント**: この一連の流れを `/research` Skill としてまとめてあります。次の章で、こうした自動化の仕組みをさらに深掘りします。
