# Chapter 5: Agent Teams - 複数エージェントを指揮する

## Subagents のおさらい

Vol.1 で学んだ通り、Claude Code は複雑なタスクを受けると **Subagent（子エージェント）** を自動生成して並列処理します。

例えば「このリポジトリのすべての .md ファイルの内容を要約して」と頼むと、Claude Code は内部で複数の Subagent を立ち上げ、各ファイルを並列で処理します。

## Agent Teams とは？

Agent Teams は、Subagents をさらに発展させた考え方です。

| | Subagents | Agent Teams |
|---|---|---|
| 役割分担 | なし（同じタスクを分割） | あり（リサーチャー、アナリスト等） |
| 指揮 | Claude Code が自動で管理 | ユーザーが構造を指示できる |
| 連携 | 独立して動く | 情報を共有・統合する |

### イメージ

```
あなた（指揮者）
  ├── エージェントA: 「市場規模を調べて」
  ├── エージェントB: 「競合3社を調べて」
  └── エージェントC: 「技術トレンドを調べて」
          ↓
    結果を統合してレポート作成
```

## Agent Teams を活用する指示の出し方

Agent Teams を効果的に使うには、**タスクの分解と指示の構造化** が重要です。

### 悪い例

```
EV業界について調べてレポートにまとめて
```

→ Claude Code は1つのタスクとして順番に処理する。時間がかかる。

### 良い例

```
EV業界のレポートを作りたいです。以下の3つの調査を並列で進めてください:

1. EV市場の市場規模と成長率（2020-2024年）
2. 主要メーカー（テスラ、BYD、トヨタ）の戦略比較
3. バッテリー技術の最新動向

各調査結果は output/ に個別ファイルで保存してください。
全て完了したら、結果を統合して分析してください。
```

→ Claude Code は3つの Subagent を並列で走らせ、効率的に処理する。

### 指示のポイント

1. **タスクを明確に分ける**: 「3つの調査を並列で」
2. **各タスクの成果物を定義する**: 「個別ファイルで保存」
3. **統合のステップを指示する**: 「完了したら統合して」

## タスク分解の考え方

業界研究を Agent Teams 向けに分解する例:

```
業界研究レポート
├── 調査フェーズ（並列実行可能）
│   ├── 市場概要の調査
│   ├── 主要プレイヤーの調査
│   ├── 技術トレンドの調査
│   └── 規制・リスクの調査
├── 分析フェーズ（調査完了後）
│   └── 調査結果の統合と分析
└── レポートフェーズ（分析完了後）
    └── レポート生成
```

**並列にできるもの**と**順番にやるべきもの**を区別するのがコツです。

### 並列にできる = 互いに依存しない

- 市場規模の調査と技術トレンドの調査 → **並列OK**（互いに無関係）
- 調査と分析 → **順番が必要**（調査結果がないと分析できない）

## 実際に並列調査を体験する

以下のように指示してみましょう:

```
以下の2つの調査を並列で実行してください:
1. curriculum/ フォルダ内の全ファイルの概要を一覧にする
2. .claude/commands/ フォルダ内の全Skillの概要を一覧にする

それぞれ別のファイルに出力してください。
```

Claude Code がどのように処理を分担するか観察してください。

> **ポイント**: Agent Teams の本質は「プログラミング」ではなく「タスク設計」です。何をどう分割すれば効率的か考える力が、Claude Code を使いこなす鍵になります。
