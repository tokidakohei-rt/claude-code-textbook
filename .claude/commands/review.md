$ARGUMENTS で指定されたファイルまたは直前の出力をレビューし、フィードバックを返してください。

## 評価観点

1. **構成**: 論理的な流れになっているか。見出し・段落の組み立ては適切か
2. **情報の質**: 根拠のある情報を使っているか。出典や具体的な数字はあるか
3. **具体性**: 抽象的な表現で終わっていないか。事例・数値・比較が含まれているか
4. **読みやすさ**: 見出し・箇条書き・強調が適切に使われているか
5. **過不足**: 重要な情報の抜け漏れはないか。冗長な部分はないか

## フィードバックのルール

- **良い点を先に2つ以上伝える**（「ここがいい！」「この部分が効果的」）
- **改善点は具体的に提案する**（「〇〇を追加すると説得力が増す」「この段落は△△に分割するとわかりやすい」）
- 改善提案は **最大3つ** に絞る（多すぎると行動に移せない）
- 点数やランクはつけない
- 前向きなトーンで締める（「ここを直せばかなり良くなります！」）

## 出力フォーマット

```
## レビュー結果

### いいね！
- [良い点1]
- [良い点2]

### もっと良くなるポイント
1. [改善提案1: 具体的なアクション]
2. [改善提案2: 具体的なアクション]
3. [改善提案3: 具体的なアクション]（あれば）

### 次のアクション
→ [最も効果の高い改善から取り組むための一言アドバイス]
```
