# 最終演習: 論文要約スライドを作る

## ゴール

`paper/` に置いた論文を要約し、`output/slides.md` にMarp形式のスライドを生成する。

## 手順

### Step 1: 論文を準備する
`paper/` ディレクトリに論文ファイルを配置してください。
- PDFでもMarkdownでもテキストでもOK
- サンプル論文 `paper/kahneman-tversky-1974.pdf` がリポジトリに同梱されています

### Step 2: 論文を読み解く
Claude Codeに論文を分析してもらいましょう。

おすすめの依頼方法:
```
paper/ にある論文を読んで、以下をまとめて:
1. タイトル・著者・年
2. 研究の背景と目的
3. 手法の概要
4. 主要な結果（具体的な数値があれば含む）
5. 考察と限界
6. 今後の展望
```

### Step 3: スライド構成を設計する
要約結果をもとに、スライドの構成（アウトライン）を考えましょう。
`templates/slide-template.md` を参考にしてください。

### Step 4: スライドを生成する
Claude Codeにスライドを作ってもらいましょう:

```
要約結果をもとに、output/slides.md にMarp形式のスライドを作って。
templates/slide-template.md のフォーマットに従って。
8-12枚程度で。
```

### Step 5: レビューと修正
生成されたスライドを確認し、修正を依頼しましょう:

```
output/slides.md を確認して。以下の観点でレビューして:
- 内容の正確性
- 1スライド1メッセージになっているか
- 箇条書きが多すぎないか
必要なら修正して。
```

## 学んだスキルの総動員

この最終演習では、ここまでの全チャプターのスキルを使います:

| スキル | 使う場面 |
|-------|---------|
| ファイル読み取り (Ch.1-2) | 論文の読み込み |
| Rules (Ch.3) | CLAUDE.md による出力制御 |
| Skills (Ch.4) | /report コマンドの活用 |
| Subagents (Ch.5) | 論文の並列分析 |
